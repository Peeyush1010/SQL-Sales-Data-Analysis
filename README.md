# SQL Sales Data Analysis

## Project Overview
This project is based on SQL data analysis using a sales dataset.  
The main goal of this project was to learn how SQL is used for filtering, aggregation, sorting, and solving basic business problems using real-world sales data.

I performed different SQL operations to analyze the dataset and generate useful business insights.

---

## Tools Used
- SQL
- SQLite
- DB Browser for SQLite
- GitHub

---

## Dataset
I used the Sample Superstore sales dataset in CSV format and imported it into SQLite database for analysis.

---

## Tasks Performed

### 1. Data Import
- Imported CSV dataset into SQLite database
- Created table for sales data

### 2. Data Exploration
- Viewed sample records
- Checked table structure using PRAGMA
- Counted total rows in dataset

### 3. Filtering Using WHERE Clause
Performed filtering based on:
- Region
- Category
- Sales
- Profit

### 4. Aggregation Using GROUP BY
Calculated:
- Total sales by category
- Average sales by region
- Total quantity sold

### 5. Sorting and Limiting
- Found top selling products
- Identified top categories by sales

### 6. Business Queries
- Top customers by sales
- Monthly sales trends
- Duplicate product entries

### 7. Data Validation
- Checked null values
- Verified total records
- Counted distinct categories

---

## SQL Concepts Used
- SELECT
- WHERE
- GROUP BY
- ORDER BY
- LIMIT
- SUM()
- AVG()
- COUNT()
- HAVING

---

## Key Insights
- Technology category generated high sales.
- West region had strong sales performance.
- Some products were sold repeatedly.
- A few customers contributed major revenue.
- Some transactions resulted in negative profit.
